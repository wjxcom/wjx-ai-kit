# 推荐结果质量评估框架与 UX 审查标准

> CMP-8 | 问卷星 AI 推荐引擎 — 输出质量保障
> 版本: v1.0 | 日期: 2026-03-20
> 产出方: 首席产品官（因质量评估师未按时交付，由 CPO 直接完成）

---

## 1. 推荐质量评分模型

### 1.1 评分维度概览

推荐引擎每次输出 Top-5 模板 + 推荐理由，需从以下 **4 个维度** 综合评估质量：

```
推荐质量综合分 Q = w₁·R + w₂·D + w₃·E + w₄·T

├── R: 相关性 (Relevance)          权重 w₁ = 0.40
├── D: 多样性 (Diversity)          权重 w₂ = 0.20
├── E: 理由质量 (Explanation)      权重 w₃ = 0.25
└── T: 时效性 (Timeliness)         权重 w₄ = 0.15
```

### 1.2 维度一：相关性评分 R (权重 40%)

评估推荐结果与用户需求的匹配程度。

| 子维度 | 权重 | 评分方法 | 分值范围 |
|--------|------|----------|----------|
| 行业匹配度 | 35% | 推荐模板行业标签 ∩ 用户行业 的比例 | 0-100 |
| 需求匹配度 | 40% | 语义相似度（用户 query embedding ⊗ 推荐模板 embedding）平均分 × 100 | 0-100 |
| 历史偏好匹配度 | 25% | 推荐模板类型与用户历史使用类型的 Jaccard 相似度 × 100 | 0-100 |

**计算公式：**

```
R = 0.35 × IndustryMatch + 0.40 × QueryMatch + 0.25 × HistoryMatch

IndustryMatch = |{t ∈ Top5 : user.industry ∈ t.industry_tags}| / 5 × 100
QueryMatch = mean(cosine_sim(query_emb, t_emb) for t in Top5) × 100
HistoryMatch = |used_categories ∩ recommended_categories| / |used_categories ∪ recommended_categories| × 100
```

### 1.3 维度二：多样性评分 D (权重 20%)

避免推荐结果高度同质化，确保用户有多元选择。

| 子维度 | 权重 | 评分方法 | 分值范围 |
|--------|------|----------|----------|
| 类别多样性 | 50% | Top-5 中不同类别数 / 5 × 100 | 0-100 |
| 模板差异性 | 50% | Top-5 模板两两余弦距离的均值 × 100 | 0-100 |

**计算公式：**

```
D = 0.50 × CategoryDiversity + 0.50 × TemplateDissimilarity

CategoryDiversity = unique_categories(Top5) / 5 × 100
TemplateDissimilarity = mean(1 - cosine_sim(t_i, t_j) for i<j in Top5) × 100
```

**质量标准：**
- D ≥ 60：多样性良好
- D ∈ [40, 60)：可接受但偏单一
- D < 40：多样性不足，需检查融合排序的去重逻辑

### 1.4 维度三：推荐理由质量 E (权重 25%)

评估推荐理由是否清晰、具体、可信。

| 子维度 | 权重 | 评分方法 | 扣分规则 |
|--------|------|----------|----------|
| 具体性 | 35% | 理由是否包含具体信息（行业名、使用人数、匹配维度） | 泛泛而谈（如"适合您"）每条扣 20 分 |
| 准确性 | 40% | 理由中的事实声明是否可验证 | 虚假声明（如错误的使用人数）每条扣 30 分 |
| 可读性 | 25% | 理由长度 10-50 字，无重复措辞 | 过短(<10字)扣 15 分；过长(>80字)扣 10 分；重复措辞扣 10 分 |

**计算公式：**

```
E_per_item = max(0, 100 - 各项扣分之和)
E = mean(E_per_item for item in Top5)
```

**推荐理由模板规范：**

| 主导策略 | 理由模板 | 示例 |
|----------|----------|------|
| 语义匹配 | "与您的需求「{query_excerpt}」高度匹配" | "与您的需求「员工满意度调查」高度匹配" |
| 基于内容 | "基于您常用的{category}类问卷推荐" | "基于您常用的市场调研类问卷推荐" |
| 协同过滤 | "同行业{industry}用户的热门选择" | "同行业教育用户的热门选择" |
| 热度加成 | "已被 {count}+ 用户使用" | "已被 2,300+ 用户使用" |

### 1.5 维度四：时效性评分 T (权重 15%)

评估推荐结果是否反映最新的模板库和用户偏好。

| 子维度 | 权重 | 评分方法 | 分值范围 |
|--------|------|----------|----------|
| 模板新鲜度 | 50% | Top-5 中含最近 30 天更新/新增模板的数量 / 5 × 100 | 0-100 |
| 画像时效性 | 50% | 用户画像最后更新时间 ≤ 24h 得 100；≤ 7d 得 70；>7d 得 30 | 30-100 |

### 1.6 综合评分与质量等级

```
Q = 0.40×R + 0.20×D + 0.25×E + 0.15×T
```

| 等级 | 分数范围 | 标签 | 行动 |
|------|----------|------|------|
| A | 80-100 | 优秀 | 无需干预 |
| B | 65-79 | 良好 | 监控，定期优化 |
| C | 50-64 | 一般 | 需排查低分维度并优化 |
| D | 0-49 | 需改进 | 立即排查，考虑降级到热门推荐 |

## 2. UX 体验审查清单

### 2.1 推荐结果展示方式

| 审查项 | 最佳实践 | 检查标准 | 通过/不通过 |
|--------|----------|----------|-------------|
| 推荐卡片布局 | 每个推荐模板以卡片形式展示，含标题 + 类别标签 + 推荐理由 + 预览入口 | 卡片信息完整，层次分明 | |
| 推荐数量 | 默认展示 5 个，支持"查看更多"展开 | 不超过 5 个首屏推荐，避免选择过载 | |
| 排序标识 | 不暴露算法分数，但通过排列顺序暗示推荐强度 | 第一个推荐位视觉权重最大 | |
| 加载状态 | 推荐计算期间展示骨架屏（Skeleton） | 不使用旋转加载图标，骨架屏与最终布局一致 | |
| 空状态 | 无推荐结果时展示热门模板或分类入口 | 永不展示空白页面 | |
| 响应时间 | 推荐结果 ≤ 500ms 返回 | 超过 500ms 先展示缓存结果，异步更新 | |

### 2.2 推荐理由表达规范

| 规范项 | 标准 | 示例 |
|--------|------|------|
| 长度 | 10-50 字（中文） | "基于您的教育行业背景，推荐此课程评估模板" |
| 语气 | 客观陈述，避免推销感 | ✅ "与您的需求高度匹配" ❌ "这是最好的模板！" |
| 信息量 | 至少包含 1 个匹配维度（行业/需求/历史） | "2,100+ 教育行业用户使用" |
| 个性化 | 包含用户相关信息时标注来源 | "根据您最近创建的满意度调查推荐" |
| 重复规避 | Top-5 理由不应出现重复句式 | 5 条理由的开头和结构应有变化 |

### 2.3 用户反馈机制设计

```
┌──────────────────────────────────────────┐
│ 推荐模板卡片                              │
│ ┌──────────────────────────────────────┐ │
│ │ [模板标题]          [预览] [使用]    │ │
│ │ 推荐理由：...                        │ │
│ │                                      │ │
│ │ [👍 有用]  [👎 不相关]  [🚫 不感兴趣] │ │
│ └──────────────────────────────────────┘ │
└──────────────────────────────────────────┘
```

| 反馈类型 | 触发动作 | 数据用途 |
|----------|----------|----------|
| 👍 有用 | 正反馈信号 +1 | 增强该策略/类型的推荐权重 |
| 👎 不相关 | 负反馈信号 -1 | 降低该类型推荐权重；弹出可选原因："行业不匹配" / "题型不合适" / "其他" |
| 🚫 不感兴趣 | 黑名单该模板 | 30 天内不再推荐该模板；记录用户不感兴趣的类型 |
| 点击预览 | 隐式正反馈 +0.5 | 模板获得更多曝光加分 |
| 使用模板 | 强正反馈 +3 | 协同过滤矩阵更新 |

### 2.4 推荐失败时的降级体验

当推荐引擎不可用或结果质量分 < 50 时的降级策略：

| 降级层级 | 触发条件 | 展示内容 | 用户感知 |
|----------|----------|----------|----------|
| L1 缓存降级 | 接口超时或 5xx | 展示用户最近一次的推荐结果缓存 | 用户无感知 |
| L2 热门降级 | 缓存过期 or 质量分 < 50 | 展示用户所在行业的热门 Top-5 | "为您推荐热门模板" |
| L3 分类降级 | 热门数据不可用 | 展示模板分类入口，引导手动浏览 | "浏览模板分类" |
| L4 搜索降级 | 全部不可用 | 展示搜索框，引导用户主动搜索 | "搜索您需要的模板" |

## 3. 合规与偏见检查

### 3.1 推荐系统偏见识别

| 偏见类型 | 描述 | 检测方法 | 缓解措施 |
|----------|------|----------|----------|
| 热门偏见 (Popularity Bias) | 高使用量模板获得过多曝光，新模板难以被推荐 | 计算推荐列表中高热度模板占比；若 Top-5 均为 Top-50 热门模板，触发告警 | 新模板探索加成（7 天窗口 +15%）；融合排序中限制热门模板最多占 3/5 |
| 行业偏见 (Industry Bias) | 某些行业模板丰富导致推荐质量不均 | 按行业分组计算推荐质量分 Q，检查行业间方差 | 模板稀缺行业启用语义匹配为主策略；标记模板密度低的行业并优先补充模板 |
| 位置偏见 (Position Bias) | 用户倾向于点击列表顶部，导致 Top-1 数据过强 | 对比不同位置的实际 CTR 与基准 CTR | 定期做位置随机化 A/B 测试来校正位置权重 |
| 反馈循环偏见 | 被推荐 → 被使用 → 更多推荐，马太效应 | 追踪模板推荐集中度（基尼系数）；若 20% 模板占据 80% 推荐曝光，触发告警 | 定期在推荐列表中注入探索性推荐（1-2 个随机优质模板） |

### 3.2 数据隐私合规要求

| 合规项 | 要求 | 实施方式 |
|--------|------|----------|
| 数据最小化 | 仅收集推荐所需的最少用户数据 | 推荐系统仅访问：行业、历史模板使用记录（匿名化）、当前查询。不访问问卷内容和回收数据 |
| 用户控制权 | 用户可关闭个性化推荐 | 设置页面提供"关闭个性化推荐"开关，关闭后展示行业热门推荐 |
| 数据留存 | 推荐行为日志有明确留存期限 | 推荐日志保留 180 天后自动清除；用户画像数据在账号注销后 30 天内删除 |
| 数据隔离 | 推荐数据不用于其他业务目的 | 推荐系统数据独立存储，不与广告、营销系统共享 |
| 合规审计 | 定期审查数据访问和使用 | 季度合规审计；数据访问日志保留 1 年 |

### 3.3 推荐透明度要求

| 透明度层级 | 面向对象 | 实施方式 |
|------------|----------|----------|
| L1 基本透明 | 所有用户 | 每条推荐附带理由文案，说明推荐依据（行业/历史/需求匹配） |
| L2 可查询 | 感兴趣的用户 | 点击"为什么推荐这个？"展开详细匹配信息（匹配维度、匹配度百分比） |
| L3 可控制 | 高级用户 | 提供推荐偏好设置：可调整推荐偏好权重（更侧重行业 vs 更侧重创新） |
| L4 审计级 | 内部合规 | 完整推荐决策日志：每次推荐的输入特征、策略权重、候选集、最终排序 |

## 4. 评估执行方案

### 4.1 自动化质量监控

```
每次推荐请求
    │
    ▼
┌─────────────────┐
│ 计算 Q 质量分    │ 实时，随推荐结果一起计算
└────────┬────────┘
         │
    ┌────┴────┐
    │ Q ≥ 50? │
    ├─ 是 ────│──→ 正常返回，Q 分记入监控日志
    └─ 否 ────│──→ 触发降级策略 + 告警
              │
              ▼
┌─────────────────┐
│ 每日聚合报告     │ 离线，每日凌晨
│ - 平均 Q 分趋势  │
│ - 各维度分布     │
│ - 低分推荐 Top-10│
│ - 偏见指标检查   │
└─────────────────┘
```

### 4.2 人工抽检标准

| 抽检频率 | 样本量 | 检查内容 | 负责人 |
|----------|--------|----------|--------|
| 每周 | 50 次推荐结果 | 相关性人工评分 vs 算法评分对比 | 产品团队 |
| 每月 | 10 个用户深度访谈 | UX 体验、理由可理解性、满意度 | UX 团队 |
| 每季度 | 全量偏见审计 | 基尼系数、行业方差、反馈循环检测 | 数据团队 |
